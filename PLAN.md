# План переформатирования лендинга: от "всё" к GetCourse

## Контекст

**Текущая проблема:**
- Начинающий техспец
- Сложный проект (Tilda + 1С + маркировка) → много времени, мало денег
- Лендинг позиционирует "я делаю всё" → привлекает неправильные проекты

**Выбранная стратегия:**
- **Ниша:** GetCourse + интеграции (Telegram-боты, платежи)
- **Модель:** проект (5–15k за 3–5 дней) + постоянная поддержка (3–5k/месяц)
- **Цель:** 5 постоянных клиентов = 25k/месяц пассивного дохода за 5–10 часов работы
- **Фокус:** GetCourse = 80% работы, остальное редко

## Текущий анализ `index.html`

**Что менять:**
- Услуги: убрать общие пункты, выделить GetCourse
- Процесс: показать стандартный цикл (консультация → настройка → поддержка)
- Кейсы: один есть (Tilda), добавить плейсхолдеры для GetCourse примеров
- Контакты: оставить как есть

## Декомпозиция: список подзадач (15–30 мин каждая)

### Фаза 1: Переформатирование услуг (45 мин)

**Задача 1.1:** Переписать секцию "Услуги" — выделить GetCourse как основное (15 мин)
- Текущие 4 карточки → переделать на:
  - GetCourse + интеграции (основная)
  - Telegram-боты (подробнее)
  - Интеграции и автоматизация (упрощённо)
  - Аудит и поддержка (для постоянных)
- **Проверка:** открыть браузер, видны ли новые названия и описания

**Задача 1.2:** Добавить подзаголовок в hero — боли GetCourse клиентов (15 мин)
- После основного заголовка добавить маркеры:
  - 🔴 Ученик оплатил — где его доступ? (ждёт часами)
  - 🔴 20 вопросов в день в Telegram (не получаешь ответы)
  - 🔴 Не видишь статистику заработков (разбросано по системам)
- **Проверка:** на мобильной резолюции читается хорошо, не обрезается

**Задача 1.3:** Обновить процесс — изменить на стандартный цикл GetCourse (15 мин)
- Изменить 4 этапа:
  - Консультация 20 мин (как теперь)
  - Оценка: план работ + стоимость (как теперь)
  - Настройка GetCourse: 3–5 дней (изменить срок)
  - Передача + предложение поддержки (новое)
- **Проверка:** срок настройки = 3–5 дней, передача = на дне 5

### Фаза 2: Обновление кейсов (30 мин)

**Задача 2.1:** Улучшить существующий кейс Tilda (10 мин)
- Оставить "OLESIA YUHOVA" как сложный пример
- Добавить пометку: "премиум-проект, маркировка + 1С"
- **Проверка:** явно понятно что это сложный кейс

**Задача 2.2:** Добавить 2 плейсхолдера кейсов GetCourse (20 мин)
- Кейс 2: "Online-школа [PLACEHOLDER: название]"
  - Стек: GetCourse, Telegram-бот, CloudPayments
  - Задача: автоматизировать выдачу доступа и ответы на вопросы
  - Результат: [PLACEHOLDER: экономия часов/месяц]
- Кейс 3: аналогично
- **Проверка:** плейсхолдеры видны, готовы к заполнению

### Фаза 3: Обновление контактов и форм (30 мин)

**Задача 3.1:** Добавить "Поддержка" в контактную информацию (10 мин)
- В секции контактов добавить пункт:
  - 💚 Постоянная поддержка: 3–5k/месяц, я слежу чтобы всё работало
- **Проверка:** видно в контактах

**Задача 3.2:** Переделать форму консультации — добавить чекбокс поддержки (20 мин)
- После поля "Задача" добавить:
  - Чекбокс: "Интересует постоянная поддержка? (+3–5k/месяц)"
  - Если отмечено → отправить в заявке флаг
- **Проверка:** форма валидируется, отправляет данные с флагом

### Фаза 4: Быстрые оптимизации (20 мин)

**Задача 4.1:** Добавить meta description и OG теги (10 мин)
- `<meta name="description" content="Максим помогает владельцам курсов на GetCourse избежать хаоса с платежами и доступом. Бесплатная консультация, быстрая настройка, постоянная поддержка.">`
- OG теги для соцсетей
- **Проверка:** PageSpeed показывает, в Facebook Sharing Debugger видны OG

**Задача 4.2:** Добавить favicon (10 мин)
- Добавить строку: `<link rel="icon" href="data:image/svg+xml,<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100"><text y=".9em" font-size="90">🚀</text></svg>">`
- **Проверка:** иконка видна во вкладке браузера
